macOS utility

Windows-style switching for macOS, driven by Cmd+Tab.

ApSwitcher replaces app-only switching with real window switching. You move across the windows you actually have open, keep previews visible, and land on the exact window you meant to open.

Why it exists

macOS app switching is app-centric. ApSwitcher is window-centric.

Switch the way Windows users expect

The switcher walks across actual windows. If you have two browser windows and one terminal, each one gets its own slot.

Use recent-window order

The first jump goes to the previous window, not an arbitrary app. That keeps fast back-and-forth workflows intact.

Stay practical about macOS limits

Accessibility powers hotkeys and focus. Screen Recording powers previews. The app stays small and focused instead of pretending those constraints do not exist.

How it works

A thin macOS runtime with most of the logic extracted into testable code.

What you get

  • Cmd+Tab and Cmd+Shift+Tab navigation
  • Arrow key navigation while the overlay is open
  • Window previews when Screen Recording is available
  • Menu bar app with a minimal exit path

What permissions are required

  • Accessibility for global hotkeys and focus changes
  • Screen Recording for live thumbnails

Install

Download, grant permissions once, and use it.

01

Download the latest release

Grab the notarized DMG from GitHub Releases and move the app into Applications.

02

Approve permissions

Grant Accessibility first, then Screen Recording if you want thumbnails.

03

Use Cmd+Tab

Keep Command pressed to navigate, release it to focus the selected window.
